桥介数物岗位需求调研

运动控制算法工程师岗位分析

2026-06-22
2026-06-22

运动控制算法工程师（实习）岗位深度分析

岗位来源：飞书招聘（f0exxg5fp6u.jobs.feishu.cn）
目标公司：桥介数物（深圳）科技有限公司（BridgeDP）

一、岗位定位

运动控制算法工程师（实习）是桥介数物最核心的实习岗位，也是机器人工程专业背景最能直接变现的入口。桥介数物的核心业务是”通用机器人小脑算法”——即为四足、双足、轮足机器人提供运动控制解决方案。

二、岗位职责推断

2.1 核心职责

职责模块 具体内容 对应技术
MPC 控制器开发与调试 基于 OCS2 框架，维护和优化非线性 MPC 足式机器人控制器 C++17、OCS2、qpOASES、Eigen
WBC 全身控制 将 MPC 输出的足底力/轨迹分解为关节力矩指令 Hierarchical QP、零空间投影、任务优先级
状态估计 融合 IMU + 关节编码器数据，实时估计机器人质心状态 EKF、互补滤波、线性卡尔曼滤波
步态规划与参数调试 调整步频、步幅、抬腿高度、足端轨迹等参数 task.info、reference.info 配置
仿真验证 在 Gazebo/Isaac Sim 中验证控制算法 ROS1、Gazebo、Rviz
实机部署与调试 将算法部署到四足/双足机器人实机，处理 Sim2Real Gap 状态估计延迟补偿、传感器噪声处理

2.2 进阶职责（加分项）

  • 参与**桥介引擎（BridgeDP Engine）**的动作开发平台建设
  • 参与CWM 数据工厂的跨本体运动数据生产管线
  • 将 MPC 方案与 RL 方案做工程整合（混合控制架构）

三、技术要求深度剖析

3.1 硬性要求（面试门槛）

技术领域 要求程度 桥介代码中的直接体现
C++ ⭐⭐⭐⭐⭐ 77% 代码为 C++，大量模板、智能指针、Eigen 矩阵运算
ROS ⭐⭐⭐⭐ launch 文件、topic 通信、rviz 可视化
机器人动力学 ⭐⭐⭐⭐ 质心动力学模型、单刚体模型、多体动力学
最优控制/MPC ⭐⭐⭐⭐⭐ 非线性 MPC 是核心架构，依赖 OCS2
CMake/编译工具链 ⭐⭐⭐ catkin build、依赖管理
Linux 开发环境 ⭐⭐⭐ Ubuntu 20.04/22.04，Shell 脚本

3.2 源码技能映射

桥介开源项目 hunter_bipedal_control 的代码结构直接反映了岗位所需技能：

技能映射关系：

  • 看懂 legged_control/mpc/ → 需要最优控制 + C++ 模板元编程
  • 看懂 legged_control/wbc/ → 需要多体动力学 + 优化理论
  • 看懂 legged_control/estimation/ → 需要状态估计 + 传感器融合
  • 跑通 legged_gazebo/ → 需要 ROS + Gazebo + CMake

四、准备策略

必做事项（2-3 周）

优先级 任务 预期成果 面试能说什么
P0 跑通 hunter_bipedal_control Gazebo 仿真 录一段机器人稳定行走的视频 “我独立跑通了贵司的开源代码”
P0 修改步态参数 观察并分析 MPC 输出变化 “我改了步频从 1.5Hz 到 2.5Hz，发现…”
P1 画出 MPC → WBC → 关节指令的完整调用链 一张清晰的架构图 “贵司代码中 MPC 输出足底力，WBC 通过分层 QP…”
P1 理解 OCS2 的系统模型定义方式 能解释为什么用单刚体模型 “单刚体模型在计算效率与精度之间做了折中…”
