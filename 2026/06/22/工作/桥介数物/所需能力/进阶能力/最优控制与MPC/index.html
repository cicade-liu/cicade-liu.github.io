最优控制与模型预测控制（MPC）

桥介数物运动控制核心架构的理论基础——理解 OCS2 框架的必经之路

一、最优控制基础

1.1 什么是最优控制？

核心问题：给定一个动态系统，找到控制输入使得某个代价函数最小。

在足式机器人中：

  • = 机器人状态（质心位置、速度、姿态…）
  • = 控制输入（足底力、关节力矩…）
  • = 机器人动力学（SRB 模型）
  • = 跟踪误差 + 控制能耗
  • ≤ 0 = 摩擦锥、关节限位…

1.2 LQR（线性二次调节器）——最优控制的起点

LQR 是最简单的最优控制器，适用于线性系统 + 二次代价：

里程碑：手写 LQR 让倒立摆稳定。

1.3 为什么 LQR 不够用？

LQR 足式机器人的需求
线性系统 机器人是非线性的（SRB 动力学含三角函数）
无约束 需要摩擦锥、关节限位等约束
无限时域 需要有限时域的预测规划
固定反馈增益 需要根据步态实时调整

→ LQR 升级为非线性 MPC

2.4 桥介 MPC 的关键参数

参数 典型值 含义 调大效果
预测步长 N 10-20 向前看多少步 更远视，但计算更慢
控制周期 dt 5ms 多久求解一次 更频繁，但对算力要求高
Q（位置权重） diag([100,100,500]) z 方向通常加更大权重 更紧的跟踪
Q（姿态权重） diag([50,50,100]) yaw 方向通常小一些 防止偏航
R（力权重） 1e-4 × I 惩罚大的足底力 动作更”省力”但可能跟踪差
摩擦系数 μ 0.6-0.8 足底-地面摩擦 更大允许更大的切向力

面试时可以说：”我改了步频从 1.5Hz 到 2.5Hz，发现 MPC 输出的力变大，走路变快但也开始打滑——因为步频太高时摩擦锥约束被激活，控制器在跟踪和稳定之间做了折中。”

三、OCS2 框架深入理解

3.1 OCS2 是什么？

OCS2（Optimal Control for Switched Systems）是 ETH Zurich 开发的开源最优控制工具箱，专门针对切换系统（如足式机器人在不同接触模式间切换）。

核心特点：

  • 支持混合系统（连续动力学 + 离散模式切换）
  • 内置非线性 MPC求解器（SLQ/iLQR + SQP）
  • 高效的自动微分和C++代码生成
  • 是 ETH ANYmal 机器人的控制框架
