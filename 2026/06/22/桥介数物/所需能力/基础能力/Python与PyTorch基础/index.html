Python 与 PyTorch 基础

第一部分：Python 与 PyTorch 的关系与分工

1.1 各自定位

Python 和 PyTorch 是两个不同层面的东西，用一句话概括：

Python 是你的语言能力，PyTorch 是你用这门语言写出的数学工具。你会说话 ≠ 你会推导数学公式，但你必须会说话才能写公式。

工具 定位 桥介项目中的角色
Python 通用编程语言 写训练脚本、数据处理、实验管理、自动化测试
PyTorch 深度学习框架 搭建策略网络、训练 RL 模型、加载 / 保存权重

进一步解释：

  • Python 是通用编程语言，提供语法、基础数据类型、文件 I/O、循环、函数、类等基础能力。能写爬虫、后端、数据分析、脚本、AI —— 所有程序都能用 Python 写。
  • PyTorch 是基于 Python 开发的深度学习框架，底层用 C++/CUDA 实现高性能运算，上层封装 Python API 方便调用。它不是一个独立的语言，而是 Python 的一个库（import torch），类似于 numpy（数值计算）、pandas（数据分析）、requests（HTTP 请求）。

第三部分：PyTorch 核心

3.1 张量（Tensor）—— 深度学习的基础数据类型

什么是张量？

张量是 PyTorch 中统一的数据存储格式，可以理解为「支持 GPU 加速的多维数组」：

维度 名称 例子
0 维 标量 单个数字 5
1 维 向量 [1, 2, 3] —— 如 12 维关节力矩
2 维 矩阵 图片灰度图、表格数据、批量观测 [batch, obs_dim]
3 维 三维张量 单张彩色图片 [通道, 高, 宽]
4 维 四维张量 批量彩色图片 [batch, 通道, 高, 宽]

神经网络中所有输入、权重、中间特征、输出全部都是张量。张量是整个深度学习计算的基础载体。

NumPy vs Tensor 的关键区别

NumPy 数组只能跑 CPU，速度受限于 CPU 的少数核心。PyTorch 张量可以一键转移到 GPU 上，利用上千个 CUDA 核心做并行矩阵运算，速度提升几十到上百倍。

什么是 CUDA？为什么 GPU 比 CPU 快？

CUDA（Compute Unified Device Architecture）是 NVIDIA 推出的并行计算平台。它的核心思想：

  • CPU：几个到几十个大核心，擅长复杂逻辑、分支跳转、串行任务。处理大规模矩阵乘法时，只能逐个元素计算，速度很慢。
  • GPU：拥有上千个小计算核心，虽然每个核心很简单，但可以同时处理数千个数据。神经网络的本质是海量矩阵乘法 —— 这正是 GPU 的最强项。

在强化学习训练中，4096 个并行环境同时仿真，每个环境都要跑一遍策略网络。如果用 CPU，只能逐个算；用 GPU，4096 个环境的推理同时完成。

3.2 自动微分（autograd）—— 训练网络的核心

为什么需要自动微分？

训练神经网络 = 用梯度下降更新参数 = 需要对损失函数求导。

  • 传统方法：手写求导公式，对每个参数手动推导偏导数。简单网络（如线性回归）勉强能做，复杂网络（数百层、百万参数）完全不可能。
  • PyTorch 的做法：设置 requires_grad=True，PyTorch 自动跟踪该张量的所有运算，构建计算图。一行 loss.backward() 就自动算出所有参数的梯度。

原理：PyTorch 在每次张量运算时，悄悄记录下「谁参与了这次运算、运算是什么」。这构成了一张有向无环图（计算图）。调用 backward() 时，沿着计算图从后往前，用链式法则自动求出每个参数的偏导数。

3.3 神经网络模块（torch.nn）

torch.nn 提供了搭建神经网络所需的所有预制组件，像一个「乐高工具箱」：

类别 常用模块 用途
网络层 nn.Linear、nn.Conv2d、nn.LSTM 全连接、卷积、循环层
激活函数 nn.ReLU、nn.Tanh、nn.ELU 引入非线性（否则多层等于一层）
容器 nn.Sequential、nn.ModuleList 快速堆叠多层
正则化 nn.Dropout、nn.BatchNorm 防过拟合、加速训练

3.4 优化器与损失函数

优化器决定「如何根据梯度更新参数」，损失函数决定「当前预测和目标的差距有多大」。

训练循环的本质（每一步都在做什么）：

  1. zero_grad() —— 清空上一次迭代的梯度（否则会累积）
  2. loss.backward() —— 自动计算所有参数的梯度
  3. optimizer.step() —— 根据梯度更新参数

第六部分：常见陷阱

  1. 张量形状不匹配：torch.Size([4096]) ≠ torch.Size([4096, 1])，RL 训练中最常见的 bug。检查 dim 参数是否正确。
  2. 设备不一致：部分张量在 CPU、部分在 GPU → 报错 Expected all tensors to be on same device。确保数据和模型都在同一设备上。
  3. 梯度未清零：每个训练步需 optimizer.zero_grad()，否则梯度会累积（累加多个 batch 的梯度，导致更新错误）。
  4. no_grad() 使用不当：推理时记得用 with torch.no_grad()，否则 PyTorch 会构建计算图、消耗大量显存，且推理速度变慢。
  5. 数据归一化：RL 训练中观测需要归一化（obs = (obs - mean) / std），否则数值范围差异大 → 训练不稳定。
  6. Isaac Gym 的 GPU buffer 生命周期：注意 gymtorch.wrap_tensor 返回的张量指向 GPU buffer，不要意外 overwrite。如果后续仿真步进覆写了 buffer，之前的张量内容也会变。
  7. state_dict vs 整个模型：保存时优先用 state_dict()（只保存参数），而不是 torch.save(model)（保存整个对象，跨版本可能不兼容）。
